# Mission: Git va GitHub (jamoada ishlash uchun)

## Why (Nega)
Ishda / jamoada boshqa dasturchilar bilan birga kod ustida **qo'rqmasdan** ishlash uchun.
Maqsad — kodni yo'qotmasdan, birovning ishini buzmasdan: o'z branch'ida ishlash,
GitHub'ga push qilish, pull request ochish va merge conflict'ni hal qila olish.

## Success looks like (Muvaffaqiyat belgilari)
- Yangi loyihada Git'ni sozlash va birinchi `commit` qilish
- O'z `branch`'ida ishlab, o'zgarishlarni GitHub'ga `push` qilish
- GitHub'da **pull request** ochish, ko'rib chiqish va `merge` qilish
- Jamoadagilarning o'zgarishlarini `pull` qilib olish
- **Merge conflict** chiqsa, vahima qilmasdan hal qilish
- Aniq, tushunarli **commit message** yozish
- "Xato qildim" holatda yo'qotmasdan ortga qaytarish (undo asoslari)

## Constraints (Cheklovlar)
- Yangi boshlovchi — har bir dars **qisqa**, sodda va **amaliy** bo'lsin
- Til: **o'zbekcha**; Git buyruq va atamalari **inglizcha** qoladi
- Asosan **terminal (CLI)**, lekin VS Code bilan bog'lab tushuntirish mumkin
- Linux muhiti (Git 2.43 allaqachon o'rnatilgan)

## Out of scope (Hozircha kerak emas)
- Git internals (chuqur ichki tuzilishi: blob/tree/object modeli)
- `rebase`, `cherry-pick`, `reflog`, `submodule` kabi ilg'or mavzular (keyinroq)
- O'z Git serverini sozlash / CI-CD
