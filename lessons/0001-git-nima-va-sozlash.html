DARS 01 · GIT ASOSLARI

Git nima, nega kerak — va kompyuterni tayyorlash

Bu darsdan keyin siz Git nimaligini bir jumlada tushuntira olasiz va kompyuteringiz jamoada ishlashga tayyor bo'ladi — har bir commit aynan sizniki ekanini Git biladi.

Nega bu muhim (sizning maqsadingiz)

Jamoada bir nechta odam bitta loyiha ustida ishlaydi. Git bo'lmasa: fayllar loyiha_final_FINAL_v2.zip ga aylanadi, kim nimani buzganini bilib bo'lmaydi, va birovning ishi ustiga yozib yuborish oson. Git aynan shu muammoni yo'qotadi — shuning uchun deyarli har bir ish joyida Git talab qilinadi.

Git aslida nima qiladi?

Git — bu version control system (versiyalarni boshqarish tizimi). Sodda qilib: Git loyihangizning holatini vaqt-vaqti bilan rasmga olib (snapshot) saqlab boradi. Har bir bunday saqlangan holat — commit deyiladi.1

Har bir commit — loyihangizning to'liq holati (faqat farqi emas).2 Shu sababli istalgan vaqtda ortga qaytib, "kecha hammasi ishlayotgan edi" degan holatga bemalol borasiz. Git, qo'shimcha ravishda, har commitda kim va qachon o'zgartirganini ham yozib qo'yadi.

Tarqoq (distributed) degani

Git'da har bir dasturchining kompyuterida loyihaning butun tarixi to'liq bo'ladi. Internet bo'lmasa ham ishlayverasiz; GitHub esa shu nusxalarni bir-biriga ulaydigan umumiy markaz vazifasini bajaradi (buni 4-darsda ko'ramiz).

1-qadam: Git o'rnatilganini tekshirish

Sizning kompyuteringizda Git allaqachon bor. Terminalni oching va yozing:

git --version git version 2.43.0

Agar shunga o'xshash versiya raqami chiqsa — tayyor. (Chiqmasa, ayting, birga o'rnatamiz.3)

2-qadam: Git'ga o'zingizni tanishtirish

Git har bir commitga "muallif" sifatida sizning ism va emailingizni yozadi. Buni bir marta sozlab qo'yamiz (--global = shu kompyuterdagi barcha loyihalar uchun):3

git config --global user.name "Ismingiz Familiyangiz"
git config --global user.email "siz@example.com"
Jamoa uchun muhim nuqta

Bu yerdagi email GitHub akkauntingiz email'i bilan bir xil bo'lsin. Shunda qilgan commitlaringiz GitHub'da sizning profilingizga bog'lanadi — ish/jamoada "kim nima qildi" aniq ko'rinadi.4 (GitHub'ingiz hali bo'lmasa — hech qisi yo'q, keyinroq ochamiz va shunda moslaymiz.)

Sozlamani tekshirib ko'rish:

git config --global --list user.name=Ismingiz Familiyangiz user.email=siz@example.com
Foydali (ixtiyoriy)

Yangi loyihalarda asosiy branch nomi main bo'lishi uchun:

git config --global init.defaultBranch main
✓ Sizning g'alabangiz

Git o'rnatilgan, ishlayapti, va endi sizni taniydi. Bundan keyin qiladigan har bir commit aniq sizning nomingiz bilan yoziladi. Bu — jamoada ishlashning birinchi poydevori. 🎉

Tekshirib ko'ramiz (quiz)

Eslab javob berish — bilimni mustahkamlaydigan eng yaxshi usul. Shoshilmang, o'ylab tanlang.

SAVOL 1 Git loyihangizni qanday saqlaydi?

To'g'ri — har commit loyihaning o'sha ondagi to'liq holati (snapshot). Shuning uchun istalgan eski holatga qaytish oson.

SAVOL 2 Nega jamoada (team) Git ayniqsa muhim?

Aynan — Git ko'p kishining o'zgarishlarini yo'qotmasdan, bir-birini buzmasdan birlashtirishga imkon beradi.

SAVOL 3 git config user.email nima uchun kerak?

To'g'ri — email commit muallifini belgilaydi. GitHub email'i bilan bir xil bo'lsa, commitlar profilingizga bog'lanadi.
