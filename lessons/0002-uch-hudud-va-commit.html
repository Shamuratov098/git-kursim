DARS 02 · GIT ASOSLARI

Git'ning yuragi: 3 hudud va commit aylanasi

Bu darsdan keyin fayllaringiz Git ichida qayerda turishini bilasiz va tahrirla → add → commit aylanasini ishonch bilan bajarasiz.

Nega bu muhim

Jamoada har bir commit — bitta mantiqiy o'zgarish bo'lgani ma'qul (toza tarix). Buning siri — staging area. Uni tushunsangiz, commitlaringiz tartibli va hamkasblaringizga tushunarli bo'ladi.

Eslatma

O'tgan darsda siz haqiqiy git commit qildingiz (e6a5f3b)! Aslida fayllaringiz uchta "hudud" orqali o'tdi. Keling, o'sha sirni ochamiz. 🔍

Uchta hudud (the three areas)

Git'da fayllaringiz uchta joyda "yashaydi". Bir marta commit qilganda, ular shu yo'lni bosib o'tadi:

Ish papkasi

Working Directory
Siz tahrirlab turgan fayllar. Hozir ko'rinadigan papka.
→git add

Saralash joyi

Staging Area
Keyingi commitга tanlangan o'zgarishlar. "Oraliq savat".
→git commit

Ombor (tarix)

Repository
Doimiy saqlangan tarix — yashirin .git papkasi.
  • Working Directory — fayllarni yaratasiz va tahrirlaysiz.1
  • Staging Area — git add bilan qaysi o'zgarishlarni keyingi commitга qo'shishni tanlaysiz.
  • Repository — git commit bilan tanlanganlar doimiy tarixга yoziladi.

Commit aylanasi (har kungi ish)

Kundalik Git ishi shu oddiy aylanadan iborat:

  1. Faylni tahrirla  →  Working Directory
  2. git add <fayl> — saralash joyiga qo'y  →  Staging
  3. git commit -m "izoh" — tarixга saqla  →  Repository
  4. (keyinroq) git push — GitHub'ga yukla

Nega "saralash joyi" (staging) kerak?

Tasavvur qiling: 3 ta faylni o'zgartirdingiz, lekin ulardan faqat 2 tasi bitta mavzuga tegishli. git add bilan aynan o'sha 2 tasini tanlab, alohida, toza commit qila olasiz.2 Bu — jamoada "har commit = bitta fikr" tartibini saqlashga yordam beradi.

Sizning ko'zoynagingiz: git status va git log

  • git status — hozir fayllar qaysi hududda (yangi? add qilingan? o'zgargan?).
  • git log — shu paytgacha qilingan barcha commitlar tarixi.

Chalkashib qolsangiz — doim git status deng. U sizga nima qilishni ham aytib turadi. 🧭

Amaliyot (buni birga qilamiz)

Chatda men bilan birga ikkinchi commitingizni qilasiz. Qadamlar:

  1. git status — yangi fayl untracked ko'rinadi
  2. git add <fayl> — saralash joyiga
  3. git status — endi staged ko'rinadi (hudud o'zgardi!)
  4. git commit -m "..." — tarixга saqlanadi
  5. git log --oneline — 2 ta commit ko'rinadi 🎉

Tekshirib ko'ramiz (quiz)

SAVOL 1 git add faylni qayerdan-qayerga o'tkazadi?

To'g'ri — git add o'zgarishni ish papkasidan saralash joyiga qo'yadi. Repository'ga esa keyin git commit olib o'tadi.

SAVOL 2 Nega staging area (saralash joyi) kerak?

Aynan — staging sizga tanlash imkonini beradi: hamma narsani emas, balki bitta mavzuga oid o'zgarishlarni toza commit qilasiz.

SAVOL 3 Holat va tarixni ko'rish uchun qaysi buyruqlar?

To'g'ri — git status hozirgi holatni, git log esa commitlar tarixini ko'rsatadi. Bular sizning "ko'zoynagingiz".
