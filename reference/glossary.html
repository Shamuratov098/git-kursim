repository (repo) — loyiha ombori
Loyihangiz va uning butun tarixi saqlanadigan joy. Texnik jihatdan — loyiha papkasidagi yashirin .git papkasi. git init → repo yaratiladi
working directory (working tree) — ish papkasi
Siz hozir ko'rib, tahrirlab turgan fayllar. Git "kuzatadigan" papkaning ko'rinadigan qismi. papkadagi .py, .txt, .js fayllaringiz
staging area (index) — saralash joyi
Keyingi commitga aynan qaysi o'zgarishlar kirishini tanlaydigan "oraliq savat". git add fayllarni shu yerga qo'yadi. git add file.txt → staging'ga tushadi
commit — saqlangan holat / "nuqta"
Loyihaning ma'lum bir ondagi snapshoti: kim, qachon, nima o'zgartirgani yozib qo'yiladi. Tarixning bir "qadami". git commit -m "Login sahifasi qo'shildi"
snapshot — holat nusxasi
Git har commitda fayllaringizning o'sha ondagi to'liq holatini rasmga olgandek saqlaydi (faqat farqni emas — shuning uchun tezda istalgan holatga qaytasiz).
commit message — commit izohi
Har commitga yoziladigan qisqa tushuntirish: nima va nega o'zgardi. Jamoada juda muhim. "Fix: bo'sh email xatosi tuzatildi"
branch — shox / tarmoq
Asosiy kodga tegmasdan, alohida ishlash uchun ajratilgan "yo'l". Tugagach asosiyga merge qilinadi. git switch -c login-feature
HEAD — "men hozir qayerdaman"
Hozir qaysi branch/commitda turganingizni ko'rsatuvchi belgi (ko'rsatkich).
main / master — asosiy branch
Odatdagi asosiy branch nomi. Yangi loyihalarda main, eskilarda ko'pincha master. Ikkalasi ham shunchaki nom.
merge — birlashtirish
Bir branchdagi o'zgarishlarni boshqasiga qo'shish. git merge login-feature
merge conflict — birlashtirish ziddiyati
Ikki kishi bir joyni har xil o'zgartirsa, Git qaysi birini olishni bilmaydi va sizdan hal qilishni so'raydi. Normal holat — vahima emas.
remote — masofaviy repo
Internetdagi nusxa (masalan GitHub'da). Jamoa shu orqali kod almashadi.
origin — asosiy remote nomi
remotening odatdagi nomi — odatda sizning GitHub repongiz. git push origin main
clone — nusxa ko'chirib olish
Internetdagi repo'ni butun tarixi bilan kompyuteringizga ko'chirish. git clone https://github.com/user/repo.git
push — yuklash (yuqoriga)
Kompyuteringizdagi commitlarni remotega (GitHub'ga) jo'natish. git push
pull — tortib olish (pastga)
Remote'dagi yangi o'zgarishlarni o'zingizga olib, qo'shib qo'yish (fetch + merge). git pull
fetch — yangiliklarni keltirish
Remote'dagi o'zgarishlarni ko'rib chiqish uchun keltiradi, lekin kodingizga hali qo'shmaydi. pull = fetch + merge.
pull request (PR) — qo'shish so'rovi
GitHub'da: "mening branch'imdagi o'zgarishlarni asosiy kodga qo'shaylik" degan taklif. Jamoa ko'rib chiqadi (review) va tasdiqlaydi. Jamoa ishining yuragi.
fork — o'z nusxangizga ko'chirish
Birovning GitHub repo'sini o'z akkauntingizga nusxalash. Open source'ga hissa qo'shishda ishlatiladi.
.gitignore — e'tibor bermaslik ro'yxati
Git kuzatmasligi kerak bo'lgan fayllar ro'yxati (masalan parollar, node_modules, vaqtinchalik fayllar). *.log
commit hash (SHA) — commit "passporti"
Har commitning takrorlanmas uzun ID'si. Odatda dastlabki 7 belgisi yetarli. a1b2c3d
